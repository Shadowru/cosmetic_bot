# Voice Generation — техническая записка для следующего агента

Файл собран для следующего Claude (или любого другого dev), который будет копаться в TTS-стеке этого репозитория. Всё что ниже — выстраданный опыт реальной отладки, не теоретические рассуждения.

## TL;DR

Закадровый голос для 30-секундных YouTube Shorts генерируется на **Silero TTS** (модель `v3_1_ru`, голос `xenia`, 48kHz), синтезируется **предложение-за-предложением** с ручными паузами, и проигрывается с пониженным темпом через FFmpeg `atempo`. Сверху ложится фоновая lo-fi музыка через `amix` на 18% громкости. Главный файл — `shorts_generator.py`, функция `_generate_voiceover()` и её соседи.

---

## Стек целиком

```
Ollama JSON (5 полей: hook, block1, block2, block3, cta)
   ↓
_preprocess_tts()          — числа → num2words, аббревиатуры → транскрипция
   ↓
_synth_sentences()         — предложение-за-предложением, 120ms тишины между
   ↓
WAV (float32 → int16, 48kHz, mono)
   ↓
FFmpeg atempo (per-card)   — 0.93× для hook, 0.82× для cta
   ↓
MP3 per-card (192k, AAC)
   ↓ (5 файлов concat + crossfade)
audio_out.aac
   ↓
_mix_music()               — рандомный трек из music/, 18% volume, fade in/out
   ↓
audio_music.aac → ложится на видео в финальной сборке
```

## Почему Silero, а не edge-tts / Google TTS / OpenAI TTS

**Edge-tts отвергнут после реального бага.** Microsoft Edge TTS принимает SSML-разметку, и казалось логичным управлять паузами через `<break time="0.5s"/>`. На практике:

1. Edge-tts при оборачивании в SSML **экранирует XML-символы**.
2. Тег `<break>` оказывался в финальной audio-дорожке **прочитанным вслух**: «брейк тайм ноль пять с»
3. Никакой комбинации escape/no-escape не дало нормального результата.

**Silero выбран** потому что:
- Бесплатный, локальный, без сетевых вызовов в горячем пути
- Естественное русское произношение голосом «xenia»
- `apply_tts()` принимает чистый текст без какой-либо разметки

**OpenAI/Google/ElevenLabs** не рассматривались всерьёз потому что:
- Платные на нашем потоке (5+ шортов/день = сотни долларов в год)
- Сетевая зависимость в момент когда нам надо стабильно генерить

## Загрузка модели — обязательно ленивая

Silero весит ~50MB, при первом запуске качается через `torch.hub.load()`. Загрузка занимает 30+ секунд. **Никогда** не вызывай это на старте процесса бота — он будет молча зависать в импорте.

```python
_silero_model = None  # глобальная переменная

def _get_silero():
    global _silero_model
    if _silero_model is None:
        import torch
        warnings.filterwarnings("ignore")
        logger.info("Загружаю Silero TTS (первый запуск)...")
        _silero_model, _ = torch.hub.load(
            repo_or_dir="snakers4/silero-models",
            model="silero_tts",
            language="ru",
            speaker="v3_1_ru",
            trust_repo=True,
            verbose=False,
        )
        logger.info("Silero TTS загружена")
    return _silero_model
```

После первой загрузки модель остаётся в памяти процесса. Перезагружать её ни на что не нужно — поведение детерминистичное, голос всегда одинаковый.

Особенность: `torch.hub` иногда обновляет код модели из snakers4/silero-models. Если поведение внезапно поменялось — проверь PIN в `torch.hub` (его нет в коде, можно добавить через `trust_repo=True` + commit hash, если нужно).

## Числа и аббревиатуры — критический препроцессинг

Silero **не выговаривает**:
- Цифры: `3` → молча проглатывает или произносит как «триномер»
- Латинские аббревиатуры: `SPF` → «спф», `RF` → «рф» (читает побуквенно по-английски)

Поэтому функция `_preprocess_tts(text)`:

```python
# Сначала аббревиатуры (важно — до замены цифр, чтобы «SPF 50» не превратилось
# в «эс-пэ-эф пятьдесят» где «эс-пэ-эф» осталось «SPF», а 50 заменилось)
_TTS_ABBREVS = [
    (r'\bSPF\s*50\b',  'эс-пэ-эф пятьдесят'),
    (r'\bSPF\s*30\b',  'эс-пэ-эф тридцать'),
    (r'\bSPF\b',       'эс-пэ-эф'),
    (r'\bUVA\b',       'ультрафиолет А'),
    (r'\bUVB\b',       'ультрафиолет Б'),
    (r'\bUV\b',        'ультрафиолет'),
    (r'\bpH\b',        'пэ аш'),
    (r'\bRF\b',        'эр-эф'),
    (r'\bLED\b',       'эль-и-ди'),
    (r'\bAHA\b',       'а-ха кислоты'),
    (r'\bBHA\b',       'бэ-ха кислоты'),
    (r'\bSLS\b',       'эс-эль-эс'),
    (r'\bPRP\b',       'пэ-эр-пэ'),
    # ... и так далее. Расширяется свободно.
]
for pattern, replacement in _TTS_ABBREVS:
    text = re.sub(pattern, replacement, text, flags=re.IGNORECASE)

# Потом цифры
from num2words import num2words
text = re.sub(r'\b\d+\b', lambda m: num2words(int(m.group()), lang='ru'), text)
```

**Порядок имеет значение.** Если поменять (сначала цифры, потом аббревиатуры), то `SPF 50` после `\b\d+\b → "пятьдесят"` станет `SPF пятьдесят` — а потом регэксп `\bSPF\s*50\b` уже не сработает потому что 50 уже нет, и мы получим «эс-пэ-эф» как fallback и потерянное «пятьдесят» где-то в середине.

При добавлении новой аббревиатуры: ставь **более длинные/специфичные паттерны раньше** (SPF 50 → раньше SPF).

## Сентенциальный синтез — лекарство от чанкинга

Если послать в `model.apply_tts()` сразу весь блок «Папулы в местах уколов на 1-3 дня — это норма, гиалуронка ещё не распределилась. Лёгкая отёчность первые 24-48 часов тоже нормальна.» — Silero внутри **разбивает на свои чанки** и склеивает их грубо. На стыках слышны щелчки, дыхание сбивается, иногда теряются последние слоги предложения.

Решение — синтезировать по предложению с явными паузами тишины:

```python
def _synth_sentences(model, text: str) -> np.ndarray:
    sentences = [s.strip() for s in re.split(r'(?<=[.!?])\s+', text.strip()) if s.strip()]
    if not sentences:
        sentences = [text.strip()]

    pause_short = np.zeros(int(SILERO_RATE * 0.12), dtype=np.float32)  # 120 мс
    chunks = []
    for i, sentence in enumerate(sentences):
        audio = model.apply_tts(
            text=sentence,
            speaker=SILERO_SPEAKER,
            sample_rate=SILERO_RATE,
            put_accent=True,    # помогает с ударениями
            put_yo=True,        # «ё» вместо «е» где уместно
        )
        chunks.append(audio.numpy())
        if i < len(sentences) - 1:
            chunks.append(pause_short)

    return np.concatenate(chunks) if chunks else np.zeros(SILERO_RATE // 4, dtype=np.float32)
```

**120 мс** подобрано на слух. Меньше — речь сливается, бо́льше — звучит как пауза для подсказки. Это «дыхательная» пауза, не риторическая.

**`put_accent=True`, `put_yo=True`** включают встроенные эвристики ударений и буквы «ё». Без них слышны характерные интонационные ошибки на словах со смещаемым ударением.

## Темп через FFmpeg `atempo`

После синтеза WAV проигрывается с **замедлением** через FFmpeg:

```python
CARD_TEMPO = {
    "hook":   0.93,  # быстрее → захват внимания на свайпе
    "block1": 0.88,
    "block2": 0.88,
    "block3": 0.85,  # к концу — медленнее для веса
    "cta":    0.82,  # CTA должен звучать спокойно/уверенно
}
```

Логика: на хуке мы боремся за остановку свайпа → ритм бодрый. К финалу нужно «отяжелить» голос, чтобы зритель воспринял CTA как _передачу ценности_, а не торопливое «спасибо что досмотрели». Профиль не симметричный — это ритмический контраст.

**Clamp** на `atempo` — FFmpeg технически принимает 0.5-100, но звук сильно искажается за пределами 0.5-2.0. Поэтому:

```python
tempo = max(0.5, min(2.0, CARD_TEMPO.get(card_type, 0.88)))
```

Это защита от случайной криво заданной константы — если кто-то поставит 0.3, FFmpeg всё равно отработает на 0.5.

## Сборка одной карточки целиком

```python
def _generate_voiceover(text: str, output_path: Path, card_type: str = "block1"):
    model = _get_silero()
    tempo = max(0.5, min(2.0, CARD_TEMPO.get(card_type, 0.88)))

    clean = _preprocess_tts(text)
    arr_f = _synth_sentences(model, clean)
    arr16 = (arr_f * 32767).astype(np.int16)

    wav_tmp = output_path.with_suffix(".wav")
    _wavfile.write(str(wav_tmp), SILERO_RATE, arr16)

    subprocess.run([
        "ffmpeg", "-y", "-i", str(wav_tmp),
        "-filter:a", f"atempo={tempo:.3f}",
        "-c:a", "mp3", "-b:a", "192k",
        str(output_path),
    ], check=True, capture_output=True)
    wav_tmp.unlink(missing_ok=True)
```

Float32 → int16 с множителем 32767 — стандартный путь сохранения в WAV. Можно было бы использовать `soundfile` или `scipy.io.wavfile`, у нас второе (см. импорты вверху файла).

Промежуточный WAV удаляется — после конвертации в MP3 он не нужен.

## Микширование с фоном

Все 5 mp3-файлов карточек склеиваются (FFmpeg concat) в одну дорожку голоса. Потом — `_mix_music()`:

```python
def _mix_music(voice: Path, output: Path):
    tracks = list(MUSIC_DIR.glob("*.mp3")) + list(MUSIC_DIR.glob("*.wav")) + list(MUSIC_DIR.glob("*.m4a"))
    if not tracks:
        shutil.copy(str(voice), str(output))
        return
    track = random.choice(tracks)
    dur = _get_audio_duration(voice)
    fade_out_start = max(0.0, dur - 2.0)
    subprocess.run([
        "ffmpeg", "-y",
        "-i", str(voice),
        "-stream_loop", "-1", "-i", str(track),
        "-filter_complex",
        (
            f"[1:a]volume=0.18,atrim=0:{dur:.3f},"
            f"afade=t=in:d=1,afade=t=out:st={fade_out_start:.3f}:d=2[bg];"
            "[0:a][bg]amix=inputs=2:duration=first:dropout_transition=2[aout]"
        ),
        "-map", "[aout]",
        "-c:a", "aac", "-b:a", "192k",
        str(output),
    ], check=True, capture_output=True)
```

Несколько решений по дороге:

- **18% громкости** — нащупано на слух. 25% уже перебивает голос, 12% едва слышно.
- **`stream_loop=-1`** — зацикливает фон если трек короче голоса. У нас все треки 1-3 мин, голос 30 сек — лупа в реальности не нужна, но защита от случайных коротких файлов.
- **`atrim=0:dur`** — обрезает фон ровно по длине голоса. Без этого `amix=duration=first` обрезает по голосу, но фон продолжает играть после `afade out`, выходя за границы видео.
- **`afade in 1s` + `afade out 2s`** — мягкие края. Без `afade in` слышен внезапный обрыв в начале (трек начинается не с тишины), без `afade out` — резкое отрезание.
- **`amix dropout_transition=2`** — плавный кроссфейд если один из источников кончается (защита, в реальности не используется).
- **graceful degradation**: если `music/` пуст — голос копируется без фона. Это для тех случаев когда деплой без музыкальных файлов.

Все треки в `music/` должны быть **lo-fi или ambient**. Любая мелодия с распознаваемой мелодией или быстрым ритмом отвлекает от голоса и убивает retention.

## Распространённые ошибки и как избегать

### Silero молчит на конкретном слове

Вероятно это аббревиатура которой нет в `_TTS_ABBREVS`. Добавь:
1. Запиши конкретное слово, которое Silero коверкает или пропускает
2. Транскрибируй на русский кириллицей с дефисами для побуквенного произношения
3. Добавь в `_TTS_ABBREVS` (порядок: длинные/специфичные паттерны выше общих)
4. Регенерируй проблемный шорт через `/generate short [proc] [tpl]`

### Странные щелчки между предложениями

Скорее всего пауза `pause_short` стала слишком короткой. Возьми обратно 120мс или больше. Прячь не на уровне int16 (тишина = 0), а через `scipy.signal.windows.hann` для мягкого затухания — но обычно это overkill.

### Темп звучит «как робот»

Атиffemppoo `atempo` ниже 0.7 начинает заметно искажать формантную структуру (голос становится «утопленным»). Никогда не ставь tempo < 0.75 в `CARD_TEMPO`. Лучше укоротить текст.

### Голос звучит «срубленным» в конце последнего предложения

Silero иногда обрывает финальную фонему. Лечится добавлением точки в конце текста (если её нет) — `apply_tts()` тогда честно отрабатывает интонацию завершения. Это можно завести через post-process: `if not text.endswith(('.', '!', '?')): text += '.'`.

### Ошибка `torch.hub` `RuntimeError: Trying to load model on CPU`

На сервере без GPU — нормально. Silero отлично работает на CPU. Если хочется ускорить:
- На 12-ядерном CPU генерация одного предложения ~1-2 сек
- На GPU (CUDA) — 0.1-0.3 сек на предложение
- Stack: добавь `device=torch.device("cuda")` в `apply_tts()` или загрузи модель с `.cuda()`

### `_synth_sentences` падает на пустом тексте

Я добавил защиту `if not sentences: sentences = [text.strip()]`. Но если Ollama вернул пустую строку (бывало) — Silero бросит исключение. Лучше проверять на пустоту _до_ синтеза в `_generate_voiceover()`.

## Что **не** стоит трогать

- **Sample rate 48000.** Меньше = заметно хуже качество. Больше — не поддерживается моделью.
- **Speaker `xenia`.** Перепробовал все женские голоса Silero — xenia самая «доверительная», что соответствует тону «подруги-косметолога» в нашем сценарии. Если меняешь — слушай в контексте полного скрипта, не отдельных фраз.
- **`put_accent=True`, `put_yo=True`.** Без них слышимо хуже. Стоят дешево, отключать незачем.
- **120мс паузы.** Тонко настроены — не менять без A/B сравнения.

---

## Что **стоит** менять, если будут гипотезы

- **`CARD_TEMPO`** — основной рычаг. Если AVD стабильно проседает на блоках, попробуй замедлить блоки на 0.05.
- **Громкость музыки** — 18% компромисс, есть смысл поднять до 22% на хуке (бодрее) и опустить до 14% на CTA (ясная подача ссылки).
- **Список треков в `music/`** — самое дешёвое улучшение. Замените lo-fi на acoustic, эффект драматический.
- **Per-card volume** для голоса — Silero даёт ровную громкость, но иногда хочется усилить хук. Можно добавить `volume=1.2` для hook-карточки.
