## Бот: ключевые технические решения

Эти решения найдены в ходе разработки — неочевидны из кода.

### TTS: Silero вместо edge-tts
edge-tts экранирует XML при оборачивании в SSML, поэтому `<break time="0.5s"/>` читается вслух как текст. Переключились на Silero (`xenia`, 48kHz). Синтез — предложение за предложением с 120ms паузами между ними (иначе речь рвётся). Темп снижен через FFmpeg `atempo`: hook 0.93×, block 0.88×, cta 0.82×.

### Числа и аббревиатуры в TTS
Silero не произносит числа и латинские аббревиатуры. Функция `_preprocess_tts()` конвертирует через `num2words(lang='ru')` и список `_TTS_ABBREVS` (SPF→эс-пэ-эф, pH→пэ аш, AHA/BHA, UV, RF, PRP и др.) перед синтезом.

### tiktok-uploader: конфликт имён
Пакет называется `tiktok_uploader`, но локальный файл `tiktok_uploader.py` его затенял. Файл переименован в `tt_uploader.py`. Если добавляешь новый файл — не называй его именем установленного пакета.

### YouTube: видимость в Studio после планирования
При нажатии «📅 Запланировать» видео сразу загружается на YouTube как `private` с `publishAt = slot`. Без этого видео в Studio не появляется до момента публикации. Telegram-публикация при этом всё равно происходит по расписанию через `publish_scheduled`.

### Склонение названий процедур
- `PROC_NAMES` — родительный падеж («лазерной шлифовки») — для YouTube-заголовков «после {proc}»
- `PROC_NAMES_NOM` — именительный падеж («лазерная шлифовка») — для отображения в интерфейсе
- `_short_title()` в `bot.py` должен использовать `PROC_NAMES`, иначе получается «после аппаратная чистка лица».

### Месячный план
2 шорта + 1 пост в день. Каждый 5-й слот шорта — general-тема (~20% контента). Процедуры и шаблоны ротируются без повторений в соседних слотах. Хранится в `content_plan.json`. Команды: `/plan [new]`, `/today`, `/today_gen`.

### TikTok: Playwright sync внутри asyncio
`tiktok-uploader` использует **синхронный Playwright API**. Если вызывать его внутри event loop бота (даже через `asyncio.to_thread()`) — падает с ошибкой `Sync API inside asyncio loop`. Решение: запускать загрузку через `subprocess.run()` в отдельном Python-процессе (`tt_upload_worker.py`), там нет event loop, всё работает.

Дополнительно: `tiktok-uploader` по умолчанию пытается запустить системный Chrome (`/opt/google/chrome/chrome`). Используем `browser="chromium"` (Playwright Chromium), его ставит `python3 -m playwright install chromium`.

### TikTok cookies: формат
`tiktok_cookies.json` — JSON, но библиотека ожидает Netscape `.txt`. Если передать `cookies=str(path)` — она пытается парсить TSV и получает пустой список → `InsufficientAuth`. Правильный способ: загрузить JSON руками и передать через `cookies_list=` (Playwright-совместимый формат).

### YouTube OAuth scope
Эволюция скоупов:
- Старт: `youtube.upload` (только загрузка)
- + `yt_stats.py`: расширили до `youtube` (полный) — для статистики
- + Analytics API: добавили `yt-analytics.readonly`
- + комменты: оказалось `youtube` НЕ хватает для `commentThreads.insert` (403 Insufficient Permission) — нужен **`youtube.force-ssl`**

Текущий состав в `youtube_auth.py:SCOPES`:
- `https://www.googleapis.com/auth/youtube.force-ssl`
- `https://www.googleapis.com/auth/yt-analytics.readonly`

При смене скоупов **удалить `yt_token.json` и переавторизоваться**, иначе `invalid_scope` или silent fail (комменты раньше валились с warn-логом, который никто не читал — теперь повышено до error).

### Авто-публикация без ревью + smart-skip
Команда `/autopublish` — toggle on/off (без аргументов, без срока). Создаёт/удаляет `autopublish.flag`. 5 cron-слотов в `main()` (`05:00, 08:00, 11:00, 14:00, 17:00 UTC` = `08, 11, 14, 17, 20 МСК`).

Перед генерацией каждого слота вызывается `shorts_generator.should_skip_slot()`. Слот пропускается если:
1. Свежих `(процедура, шаблон)` комбо осталось < 5 (растягиваем оставшиеся ~5 публикаций на бóльший срок, не начинаем повторяться)
2. Средний AVD последних 10 шортов < 22% (алгоритм охладел, не валим больше)

Логика OR: пропуск при любом из условий. Скип логируется в bot.log, TG-уведомлений нет (раздражали бы при ежедневных пропусках). Эффект: при текущих 188 комбо публикуем 5/день первые ~25 дней, потом плавный спад до 1-2/день естественным образом.

### TG-канал НЕ дублирует шорты
Флаг `CROSSPOST_SHORTS_TO_TG = False` в `bot.py`. Раньше каждый шорт постился и в YouTube, и в TG-канал — это убивало смысл подписки на канал (контент == канал). Теперь TG имеет уникальный контент: текстовые посты от `daily_generate` + lead-magnets (календари по процедурам) в закрепах. Воронка: YouTube → QR → TG-канал → видит lead-magnet → подписывается.

### Структура хука + визуальный кикер
Из YouTube Analytics видно: топ-AVD у `days` (39.6%) — этот шаблон обещает «3 дня по плану». Поэтому в `SCRIPT_SYSTEM` теперь явно требуется хук с **обещанием "3 [правила/признака/ошибки]"** — незакрытый список держит до конца. На карточках 1/3 → 2/3 → 3/3.

Параллельно: `KICKER` словарь — большое слово на хук-карточке (СТОП. / ОШИБКА. / ТРЕВОГА?), бьёт в первые 0.5s. Помогает swipe-to-watch (YT-метрика).

### Engagement-коммент ≠ ссылка
Авто-коммент после загрузки **раньше пушил ссылку на Telegram** — алгоритм YouTube это бьёт (увод с платформы). Теперь по словарю `_COMMENT_QUESTIONS` бот пишет конкретный вопрос по процедуре («У тебя были папулы после биоревитализации?»). Цель — спровоцировать ответы, дать алгоритму engagement-сигнал.

### Вопрос в block3, а не в CTA
YouTube Studio показал что **в CTA-карточке зрители уходят** в последние 5s («value over» сигнал). Перенесли engagement-вопрос **в конец block3** — на пике retention. CTA теперь чистая передача ценности: «ЗАБИРАЙ →» крупно + QR + «📸 СОХРАНИ» инструкция. Без вопросов, без «подписывайся» — ритм не провисает.

Validation: `Мифы про уход после ботокса` (`myths/botox`) получил **81.2% AVD** на 27-секундном видео — самый сильный сигнал за всю историю канала. Это базовая структура которую защищаем.

### CTA-обещание = lead-magnet в TG
Раньше говорили «полный список средств — в Telegram». Зритель приходил по QR и видел ленту шортов (тогда ещё кросспостились) или пустоту — несоответствие убивало подписки. Теперь:
1. CTA озвучка обещает **конкретный** магнит: «**календарь по дням** после {proc} — в закрепе Telegram»
2. TG-канал имеет реальный закреп с детальным календарём (генерация через `/post_magnets`)
3. Шорты больше не кросспостятся в TG (`CROSSPOST_SHORTS_TO_TG=False`)
4. После клика по «Подобрать набор» бот `@nenaugad_bot` показывает inline-кнопку «📅 Календарь в канале»

### Серии видимые зрителю
В `SERIES` (12 серий по 3 видео) каждая процедура имеет цепочку. Раньше зритель об этом не знал. Теперь на хук-карточке появляется плашка **«СЕРИЯ · N/3»** в левом верхнем углу — создаёт незакрытый сюжет и даёт причину подписаться. Helper `_series_position(procedure, template)` возвращает позицию или None. Третий элемент в каждой серии — `norm_alarm` (Норма vs Тревога) — отвечает на самый частый вопрос аудитории «is this OK?».

### Шаблон norm_alarm + Gemini fact-check
Per-procedure «Норма или тревога после X» (по совету YouTube Studio). СТРУКТУРА БЛОКОВ ЖЁСТКАЯ: block1 — норма, block2 — норма, block3 — ТРЕВОГА + микро-вопрос. Кикер «НОРМА?». Двойной hook: curiosity_gap + spike-в-конце.

**Медицинская безопасность:** после генерации каждый `norm_alarm` сценарий проходит fact-check через Gemini 3.5 Flash (`validators.py`). Gemini проверяет: правда ли «норма» это норма для конкретной процедуры, правда ли «тревога» — повод к врачу, разумны ли сроки. При `severity=fail` — регенерация (до 3 попыток), при `warn` — пропуск с warn-логом, при `ok` — тихо.

### Gemini API: proxyapi.ru native-Google путь
Нативный Google AI Studio API возвращает FAILED_PRECONDITION в РФ-регионе. Используем прокси **proxyapi.ru** — но **только их `/google/v1beta/...` путь**, не `/openai/v1`. На OpenAI-совместимом пути они дают только OpenAI-модели; Gemini есть только через native-Google формат: POST на `https://api.proxyapi.ru/google/v1beta/models/{MODEL}:generateContent?key=...`, body Google-формата `{contents: [{parts: [{text}]}], generationConfig: {...}}`. Модель `gemini-3.5-flash` доступна. Default в коде уже стоит — её. Если захочется сменить — `PROXYAPI_MODEL` в .env.

### Bot occasional silent hang (без systemd auto-restart)
27.05 бот упал в состояние `Application is stopping` без видимого Traceback в логах — провисел 9 часов, пропустил 4 из 5 авто-слотов. Запуск через `nohup` не возвращает к жизни упавший процесс. Если повторится — стоит перевести запуск на systemd с `Restart=always`. Симптом: `ps aux | grep bot.py` показывает живой процесс, но в bot.log последняя запись «Application is stopping» с давним timestamp и никаких новых job-ов. Решение немедленное — `kill -9` + `nohup python3 bot.py >> bot.log 2>&1 &` (с `>>` чтобы лог не стирался).

### Методология анализа: «старая vs новая школа»
Удобный способ оценить эффект серии правок — разделить шорты по дате до/после внедрения и сравнить avg views и avg AVD. У нас 30 старых (до 20.05) vs 16 новых (с 21.05) показали +14% views и +6 п.п. AVD. Это объективный аргумент что серия правок работала, не только субъективный. Воспроизводится одним python-скриптом по `analytics.json` — стоит держать как template для будущих эпизодов «много изменений сразу».

### Ollama дропает поля → retry в `_call_ollama`
Иногда (особенно после правок промпта где `cta` стал безвопросной фразой ценности) Ollama возвращает JSON **без какого-то поля** — обычно сливает `cta` в `block3` или просто пропускает. Раньше падало `KeyError: 'cta'` в `_build_video`. Теперь в `_call_ollama` проверяются все 5 ключей `_REQUIRED_SCRIPT_KEYS = (hook, block1, block2, block3, cta)`, и при отсутствии любого — retry до 3 раз. Если все 3 фейлятся — `ValueError` с диагностикой (какие ключи отсутствовали). Покрывает все 5 полей сразу.

### Аббревиатуры в заголовках: НЕ .capitalize()
`.capitalize()` в Python поднимает первый символ и **опускает** все остальные. Это ломало «SMAS-лифтинга» → «Smas-лифтинга» и «RF-лифтинга» → «Rf-лифтинга». Замена — функция `_first_cap()` в bot.py: `s[:1].upper() + s[1:]` (поднимает только первый, остальное не трогает). Применяется в `_short_title()` и `_send_short_review()`.

### Engagement-коммент: backfill
Когда фиксили scope (`youtube` → `youtube.force-ssl`), осталось 25 видео без комментариев. Скрипт-бэкфилл в одной команде перебрал все, проверил отсутствие комментов, поставил вопрос из `_COMMENT_QUESTIONS[procedure]` + 👇. Шаблон bash-скрипта остаётся в истории терминала на случай повторной необходимости.

### Instagram: только manual
Команда `/insta [N]` присылает топ-N видео за 14 дней (sorted by `views × AVD/100`) — как файлы для скачивания. Помечает `ig_selected_at` в queue чтобы не дублировать. `/insta reset` сбрасывает пометки. Автоматизировать постинг в IG не стали: Meta заблокирована в РФ, бан-риски instagrapi высокие, бизнес-API требует верификации.

### Чёрный список шаблонов
В `shorts_generator.TEMPLATE_BLACKLIST` — шаблоны исключённые из авто-подбора по данным retention. Сейчас там `first_time` (3 видео, avg 227 просмотров, AVD 19.4%). `content_plan.TEMPLATES` фильтрует через этот список.

### Только один systemd-юнит для бота
В `/etc/systemd/system/` может оставаться **только один** unit, запускающий `bot.py`. 28.05 нашли дубль: помимо нового `posleprocedur-bot.service` был старый `postprocedure-bot.service` (с другим написанием), оставшийся от прошлой настройки. После ребута оба стартовали, → Telegram `Conflict 409: terminated by other getUpdates request` (нельзя долго polling-ить одним токеном из двух процессов). Авто-публикация продолжала работать, но команды боту иногда уходили не в тот инстанс. Каноничный файл лежит в `post_procedure/posleprocedur-bot.service` — установка: `sudo cp post_procedure/posleprocedur-bot.service /etc/systemd/system/ && sudo systemctl daemon-reload && sudo systemctl enable --now posleprocedur-bot`. Перед установкой: `systemctl list-unit-files | grep -i bot` — убедиться что других нет.

### Downtime 28.05 (kernel update + 11h hosting outage)
Между 01:15 и 12:13 UTC сервер был off. Маркер `-- Boot ... --` в `journalctl --list-boots`, ядро сменилось `5.15.0-176` → `5.15.0-179`. Скорее всего хостер делал обслуживание узла + накатил kernel patch. Systemd `Restart=always` сработал штатно при возврате — бот поднялся через 14s после загрузки сети. Из-за дубль-юнита (см. выше) первый старт упал с `httpx.ConnectError: All connection attempts failed` (сеть ещё поднималась), второй уже прошёл. Урок: при downtime > 5min проверить `journalctl --list-boots` — был ли ребут.

## Критически важное: РРЦ-ограничение

Бренды (Holy Land, Gigi, Janssen и др.) жёстко контролируют рекомендованную розничную цену. **Демпинговать нельзя** — иначе лишат дистрибьюторского контракта.

Как конкуренты обходят РРЦ:
- **Баллы/кэшбек**: продают за 5 000 ₽, начисляют 1 000 баллов
- **GWP (Gift with Purchase)**: миниатюры в подарок вместо скидки — работает лучше
- **Наборы (Bundles)**: скидка 15% на набор — дистрибьюторы закрывают глаза
- **Промокоды блогеров**: скрытые скидки, не индексируются

## Главные находки конкурентного анализа

### Системные уязвимости рынка (незанятые ниши):
1. **Никто не обучает применению** — как применять профессиональную косметику дома, курсами, в системе
2. **Нет кейсов домашнего применения** — только салонные до/после
3. **Нет гарантий результата** — только «оригинальность товара»
4. **Нет подписочной модели** — хотя уход курсовой по природе
5. **Нет trial-kit** — барьер входа высокий (5–15 тыс. ₽ за курс)
6. **Только 1 конкурент использует рассрочку** (Industria Krasoty)

### Штампы, которые нельзя использовать как УТП:
«Профессиональная косметика для домашнего ухода» · «Салонный уход в домашних условиях» · «100% оригинальная продукция» · «Широкий ассортимент» · «Бесплатная доставка» · «Программа лояльности»

### Ключевая возможность:
Стать не «ещё одним магазином», а **брендом результата** — с протоколами, именованным экспертом, персонализацией, гарантией.
